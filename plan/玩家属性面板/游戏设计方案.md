# 玩家属性查看面板 - 游戏设计方案

## Context

当前游戏中，玩家在战斗场景和休息场景中都无法直观地查看自己的完整属性状态。现有 UI 仅有顶部血条和金币数，缺少对移动速度、角色能力、已获得加成等信息的统一展示。需要设计一个按 Tab 键打开的属性面板，让玩家随时了解自身状态。

> **注意**: 当前休息场景中 Tab 键已绑定打开背包(InventoryPanel)，新方案将统一 Tab 键行为，在所有场景打开同一个属性面板，背包功能作为面板的一个子页签整合进来。

---

## 一、功能概述

- **触发方式**: 按 Tab 键打开/关闭面板（切换式，非按住）
- **可用场景**: 战斗场景、休息场景均可使用
- **打开时行为**: 游戏暂停（与 ESC 暂停菜单一致），避免战斗中查看属性导致死亡
- **面板类型**: 居中紧凑弹窗，带半透明遮罩，内容区支持滚轮滚动
- **面板尺寸**: 紧凑设计，约占屏幕 40-50%，避免遮挡过多游戏画面

## 三、页签系统设计

面板底部有两个页签按钮：

### 页签 1: 属性页（默认页）
展示如上布局的所有属性信息。

### 页签 2: 背包页
将现有 InventoryPanel 的内容迁移到此页签下：
- 展示已购买的道具列表
- 每个道具显示图标、名称、数量、描述
- 替代原休息场景中独立的背包面板
- 背包页内容同样支持滚轮滚动

> **整合说明**: Tab 键在所有场景统一为"打开属性面板"，背包成为面板的一个子页签，交互更一致。

## 四、信息分区详细说明

### 4.1 标题栏
| 信息项 | 数据来源 |
|--------|---------|
| 角色名称 | `CharacterData.name` |

### 4.2 基础属性区
| 属性 | 数据来源 | 显示格式 |
|------|---------|---------|
| 生命值 | `GameManager._health` / `GameManager.max_health + max_health_bonus` | 进度条 + "当前/最大" |
| 移动速度 | `Player.base_speed` * (1 + `speed_boost_percent`%) * `_speed_multiplier` | 数值 + 加成百分比（如有） |
| 加速度 | `Player.acceleration` | 数值 |
| 摩擦力 | `Player.friction` | 数值 |

### 4.3 加成效果区
| 属性 | 数据来源 |
|------|---------|
| 金币 | `GameManager._coins` |
| 金币产出加成 | `GameManager.coin_spawn_rate_bonus` |
| 钻石产出加成 | `GameManager.diamond_spawn_rate_bonus` |
| 敌人频率调整 | `GameManager.enemy_spawn_rate_penalty` |
| 红钥匙 | `GameManager.red_keys_collected` |

### 4.4 角色能力区
| 信息 | 数据来源 |
|------|---------|
| 能力名称 | `CharacterAbility.ability_name` |
| 能力描述 | `CharacterAbility.ability_description`（简短一行） |

### 4.5 当前增益区（动态）
| 信息 | 数据来源 |
|------|---------|
| 增益名称 + 剩余时间 | `Player._buff_timers` 中的活跃 buff |
| 无敌星状态 | `Player._is_star_invincible` |
| 无增益时 | 显示 "当前无活跃增益" |

## 五、滚动设计

### 5.1 滚动容器
- 面板内容区使用 ScrollContainer 包裹
- 面板固定高度（约屏幕 50-60%），内容超出时自动出现滚动条
- 标题栏和底部页签栏固定不滚动，仅中间内容区滚动

### 5.2 滚动交互
| 操作 | 行为 |
|------|------|
| 鼠标滚轮（在面板区域内） | 上下滚动内容 |
| 鼠标拖拽滚动条 | 上下滚动内容 |
| 鼠标滚轮（在面板区域外） | 无反应（不误触） |

### 5.3 滚动条样式
- 自定义细滚动条（宽 4-6px），半透明
- 与面板整体深色风格一致
- 不滚动时自动隐藏，滚动时淡入

## 六、交互设计

### 6.1 键盘操作
| 按键 | 行为 |
|------|------|
| Tab | 打开/关闭面板（切换） |
| Q / E | 切换页签（属性 <-> 背包） |
| ESC | 关闭面板（面板打开时 ESC 先关面板，而非打开暂停菜单） |
| 1 / 2 | 直接跳转到对应页签 |
| 滚轮 | 滚动面板内容 |

### 6.2 鼠标操作
- 点击页签切换页面
- 点击关闭按钮或遮罩区域关闭面板
- 滚轮滚动内容

### 6.3 游戏暂停
- 面板打开时 get_tree().paused = true
- 面板关闭时恢复 get_tree().paused = false
- 处理与 ESC 暂停菜单的优先级：面板打开时 ESC 先关面板

## 七、视觉风格

### 7.1 整体风格
- 与游戏现有 UI 保持一致（参考 PauseMenu、InventoryPanel 的视觉风格）
- 面板背景: 深色半透明
- 边框: 圆角矩形，带微妙描边
- 紧凑排版，行间距适中（不宜过疏），充分利用面板空间

### 7.2 配色参考
- 面板背景: 深灰半透明
- 标题栏: 略深的背景条，角色名称白色高亮
- 分隔线: 半透明白色细线，区分各区域
- 属性名: 浅灰色
- 属性值: 白色
- 加成数值: 绿色（正值）/ 红色（负值）
- 进度条: 绿色填充（与 HPBar 一致）

### 7.3 动效
- 打开: 缩放弹出（scale 0.95 -> 1.0，持续 0.12s）
- 关闭: 缩放收回 + 淡出（持续 0.08s）
- 页签切换: 内容淡入淡出（0.1s）

## 八、实现要点

### 8.1 新增输入动作
在 project.godot 的 [input] 中添加:
- `toggle_stats` -> 绑定 Tab 键

### 8.2 场景结构
- 新建 `scenes/ui/PlayerStatsPanel.tscn` 作为独立面板场景
- 挂载到 `scenes/GameRoot.tscn` 的 GlobalUI 层级下（与现有 HPBar、CoinLabel 同级）
- 面板在所有场景中都可用

### 8.3 面板场景树结构
```
PlayerStatsPanel (Panel/ColorRect - 半透明遮罩)
├── MarginContainer
│   └── VBoxContainer
│       ├── TitleBar (HBoxContainer) -- 角色名称 + 关闭按钮
│       ├── ScrollContainer -- 内容滚动区
│       │   └── VBoxContainer
│       │       ├── 基础属性区 (VBoxContainer)
│       │       ├── 加成效果区 (VBoxContainer)
│       │       ├── 角色能力区 (VBoxContainer)
│       │       └── 当前增益区 (VBoxContainer)
│       └── TabBar (HBoxContainer) -- [属性] [背包] 页签
```

### 8.4 关键修改
| 文件 | 修改内容 |
|------|---------|
| `project.godot` | 添加 `toggle_stats` 输入映射 |
| `scripts/GameRoot.gd` | 处理 Tab 键输入，控制面板显示/隐藏和暂停 |
| `scenes/GameRoot.tscn` | 添加 PlayerStatsPanel 实例到 GlobalUI |
| `scenes/levels/RestAreaLevel.tscn` | 移除独立的 InventoryPanel，改用统一面板的背包页签 |
| 新建 `scripts/ui/PlayerStatsPanel.gd` | 面板逻辑：数据读取、UI 更新、页签切换、滚动管理 |
| 新建 `scenes/ui/PlayerStatsPanel.tscn` | 面板 UI 场景 |

### 8.5 与现有系统的集成
- 数据全部从 GameManager 和 Player 单例读取，无需新的数据流
- 接入 GameManager 的已有信号（coins_changed, health_changed）实现实时更新
- 接入 Player 的 buff 系统显示当前增益
- 替换休息场景的 InventoryPanel，统一为面板的背包页签

## 九、验证方式

1. 在战斗场景按 Tab -> 面板弹出，游戏暂停，显示正确的角色属性和当前 buff
2. 在休息场景按 Tab -> 同样弹出面板，不再打开旧背包
3. 切换到背包页签 -> 能看到已购买的道具
4. 按 ESC / 再次 Tab / 点击遮罩 -> 面板关闭，游戏恢复
5. 购买商店道具后打开面板 -> 属性数值已更新
6. 触发 buff 后打开面板 -> 当前增益区显示对应 buff 和剩余时间
7. 不同角色打开面板 -> 标题栏显示对应角色名称
8. 面板内容较多时 -> 滚轮可流畅滚动，标题栏和页签栏保持固定
9. 小屏幕/低分辨率下 -> 面板不超出屏幕，滚动正常工作
