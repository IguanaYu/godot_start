# 可视化对话编辑器插件 — 分步实现计划

## Context

对话系统 Phase 1-5 已完成（数据结构、管理器、运行时、UI、探索/Iguana/存档），测试场景验证通过。现在要实现 Phase 6：在 Godot 编辑器中可视化编辑对话图，替代目前手写代码创建对话图的方式。

目标：创建一个编辑器插件，用 GraphEdit 画节点图，保存为 .tres 文件，DialogueManager 自动加载运行。

## 关键约束

- .tscn 文件不写 uid=，引用只写 path=
- 复用现有 6 个 Resource 类（DialogueGraph/DialogueNodeData/DialogueConnection/ChoiceData/ConditionData/ActionData）
- 端口编号必须匹配运行时：CHOICE 用 port 1..N（无 port 0），CONDITION 用 port 0=true / 1=false
- 每步独立可测试，逐步增量

## 7 步拆分

### Step 1: 插件骨架 + 空面板

**新建文件**：
- `addons/dialogue_editor/plugin.cfg` — 插件配置
- `addons/dialogue_editor/plugin.gd` — EditorPlugin 入口，@tool，用 `add_control_to_bottom_panel()`
- `addons/dialogue_editor/editor/dialogue_editor.tscn` — 简单 VBoxContainer + Label

**修改文件**：
- `project.godot` — `[editor_plugins]` 添加 dialogue_editor

**验证**：编辑器底部面板出现 "Dialogue Editor" 标签页，点击可看到标签

### Step 2: GraphEdit 画布 + 工具栏

**新建文件**：
- `addons/dialogue_editor/editor/dialogue_editor.gd` — @tool，extends VBoxContainer

**更新文件**：
- `addons/dialogue_editor/editor/dialogue_editor.tscn` — 布局改为：

```
VBoxContainer
├── Toolbar (HBoxContainer): [New] [Save] [Load] | [+ Start] [+ Dialogue] [+ Choice] [+ Condition] [+ Action] [+ Sub] [+ End]
└── HSplitContainer
    ├── GraphEdit
    └── Sidebar (VBoxContainer, 宽280): "Properties" 标签
```

**验证**：底部面板显示工具栏、空画布（可拖拽缩放）、侧边栏

### Step 3: 7 种自定义 GraphNode + 节点创建

**新建文件**：
- `addons/dialogue_editor/nodes/base_dialogue_node.gd` — 基类，持有 DialogueNodeData 引用，setup() 方法，颜色方案
- `addons/dialogue_editor/nodes/start_node.gd` — 0 输入，1 输出，绿色
- `addons/dialogue_editor/nodes/dialogue_node.gd` — 1 输入，1 输出，蓝色，含 TextEdit
- `addons/dialogue_editor/nodes/choice_node.gd` — 1 输入，动态输出端口 1..N，黄色
- `addons/dialogue_editor/nodes/condition_node.gd` — 1 输入，2 输出 (True/False)，橙色
- `addons/dialogue_editor/nodes/action_node.gd` — 1 输入，1 输出，紫色
- `addons/dialogue_editor/nodes/sub_dialogue_node.gd` — 1 输入，1 输出，青色，含 target_graph_id LineEdit
- `addons/dialogue_editor/nodes/end_node.gd` — 1 输入，0 输出，红色

**节点纯 GDScript 构建 UI**（不建 .tscn，避免 uid 问题）

**更新文件**：
- `addons/dialogue_editor/editor/dialogue_editor.gd` — 按钮连接创建对应节点

**验证**：点击各按钮在画布上创建对应颜色的节点，可拖拽，端口正确，可删除

### Step 4: 节点连线 + 属性侧边栏

**新建文件**：
- `addons/dialogue_editor/editor/property_panel.gd` — @tool，编辑选中节点的属性

**更新文件**：
- `addons/dialogue_editor/editor/dialogue_editor.gd` — 连接 connection_request/disconnection_request 信号，节点选中/取消选中切换属性面板

**属性面板内容（按节点类型）**：
- DIALOGUE: speaker 下拉, text_key 输入, portrait 纹理选择
- CHOICE: 选项列表（text_key + 条件预览 + 动作数），增删按钮
- CONDITION: condition_type 下拉(10种), params 编辑, negated 复选
- ACTION: 动作列表（action_type 下拉(12种) + params），增删按钮
- SUB_DIALOGUE: target_graph_id LineEdit
- START/END: 无额外属性

**验证**：拖拽端口连线成功，点击节点显示属性面板可编辑，属性变更同步到节点数据

### Step 5: 保存/加载 .tres 文件

**新建文件**：
- `addons/dialogue_editor/editor/graph_serializer.gd` — 序列化/反序列化工具

**更新文件**：
- `addons/dialogue_editor/editor/dialogue_editor.gd` — New/Save/Load 逻辑

**流程**：
- **Save**: 收集所有 DialogueNodeData + DialogueConnection → 构建 DialogueGraph → `ResourceSaver.save()` 到 `resources/dialogue/{graph_id}.tres`
- **Load**: FileDialog 选择 .tres → `load()` 为 DialogueGraph → 清空画布 → 重建节点和连线
- **New**: 确认清空画布

**验证**：创建 Start→Dialogue→End → 保存 → 清空 → 加载 → 节点和连线恢复 → 运行游戏 DialogueManager 能加载执行该 .tres

### Step 6: 图级属性面板 + 验证

**新建文件**：
- `addons/dialogue_editor/editor/graph_properties_panel.gd` — 编辑 DialogueGraph 级别属性
- `addons/dialogue_editor/editor/graph_validator.gd` — 保存前验证

**graph_properties_panel**: graph_id, npc_id, priority, repeatable, prerequisites (graphs/quests/day/exploration/flags + AND/OR), completion effects (flags/quests/unlocks)

**graph_validator 检查项**: graph_id 非空, 至少 1 个 START 和 END, 所有节点可达, CHOICE 至少 1 个选项, CONDITION 有 true/false 连线, SUB_DIALOGUE target_graph_id 非空, node_id 无重复

**验证**：未选中节点时显示图属性，保存前自动验证

### Step 7: 润色 + 右键菜单 + 撤销

**新建文件**：
- `addons/dialogue_editor/editor/undo_manager.gd` — 简易撤销栈

**功能**：
- 右键菜单：空白处添加节点，节点上 Delete/Duplicate/Disconnect
- 快捷键：Ctrl+S 保存, Ctrl+Z 撤销, Ctrl+Y 重做, Delete 删除, Ctrl+D 复制
- 视觉：端口标签（Choice 显示选项文本, Condition 显示 True/False）, GraphEdit minimap, 网格吸附
- 状态栏：graph_id, 节点数, 连线数, 是否有未保存更改

**验证**：右键菜单可用，撤销/重做正常，快捷键生效
